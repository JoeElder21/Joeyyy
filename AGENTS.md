# APEX Chief of Staff Repository Guidance

## Purpose

This repository defines Joe Elder's professional APEX Chief of Staff agent. Keep the implementation focused on professional prioritization, project coordination, meeting preparation, decision support, and drafting.

## Non-negotiable boundaries

- APEX contains professional and career context. JEOS contains personal context. Do not combine them unless Joe explicitly requests it for the current task.
- Default to read-only analysis and previews.
- Never send a message, change a calendar, edit an external file, create a task, commit code, or perform another consequential action without explicit approval in the active conversation.
- Never expose secrets, credentials, private connector payloads, or unrelated personal information.
- Do not represent the agent as a substitute for licensed engineering, legal, medical, or financial judgment.
- Do not claim that Yaps Memory or another connector is available until its tools succeed in the active session.

## Change standards

- Preserve the approval-first design.
- Keep instructions direct, testable, and free of aspirational claims.
- Prefer small, reviewable changes.
- When changing the agent contract, update the documentation and relevant templates in the same change.
- Validate all TOML files before committing.
