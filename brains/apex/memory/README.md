# APEX Memory Boundary

The logical prefix for APEX specialist memory is `APEX::`.

- Every specialist has one unique namespace in `../agents.toml`.
- A specialist may read only mission-authorized APEX evidence.
- Private runtime memory stays in the authorized APEX command center and is never copied into this public repository.
- Proposed writes identify an APEX target, source evidence, expected state, validation, rollback, and writer lease.
- Agent 007 alone performs and reads back canonical mutations while the specialists remain in shadow stage.
- No namespace, roundtable, artifact, or raw source crosses into JEOS.

This directory records policy only; it must never become a store for live client, employer, project, lead, or connector data.
