# APEX Agent Community Protocol

## Community model

The APEX Chief of Staff is the coordinator. Specialist agents contribute bounded expertise, evidence, validation, or implementation. The coordinator owns the shared plan, assigns work, routes follow-ups, resolves conflicts, and reports the integrated result to Joe.

## Delegation packet

Every delegated task should state:

1. Outcome and definition of done.
2. Evidence and systems the agent may use.
3. Constraints, deadline, and professional-risk flags.
4. Whether the agent is advisory or the designated writer.
5. Required return format and validation evidence.

## Handoff packet

Every specialist returns:

- Status: completed, partial, or blocked.
- Findings or changes made.
- Evidence, sources, tests, and tool results.
- Assumptions and unresolved risks.
- Recommended next handoff, if any.

## Coordination rules

- Run independent work in parallel when it reduces elapsed time or improves coverage.
- Use one designated writer for each file, branch, calendar event, task record, message thread, or external record.
- Route cross-agent questions and follow-ups through the coordinator so the shared plan remains coherent.
- Reconcile disagreements with evidence. Assign a targeted verifier when evidence remains incomplete.
- Keep shared context minimal and task-specific. Never pass credentials, secrets, or unrelated private information.
- Do not claim another agent received, completed, or understood a handoff unless the active session confirms it.

## Completion report

The coordinator reports:

- Agents used and their assignments.
- Actions completed and systems changed.
- Validation performed.
- Remaining blockers or decisions.
- Joe's next move, only when one remains.
