# Brain Cadence Runbook

This runbook turns the two specialist manifests into repeatable daily, weekly, and monthly orchestration plans. It is a routing contract, not a claim that agents run continuously or on a background schedule. Agent 007 starts each cycle only in a verified runtime and supplies separate, schema-valid v2.1 packets for every specialist invocation.

## Universal cycle

1. Agent 007 confirms the owner brain and rejects mixed or unknown evidence.
2. Explicit agent selection and canonical target ownership override intent routing. Otherwise, lower numeric route precedence wins.
3. Agent 007 selects one registered mode and the minimum required artifact types for each specialist.
4. Each specialist receives only PacketGuard-validated evidence; specialists never call connectors directly.
5. Agent 007 integrates typed artifacts, resolves same-brain challenges, and holds the writer lease while every specialist remains `shadow`.
6. Any mutation is deterministic, lease-bound, read back, and paired with rollback evidence.
7. Missing evidence remains missing. A partial cycle is reported as partial and never backfilled with invented activity.

## APEX cadence

| Cadence | Order | Required decision |
|---|---|---|
| Daily | Intelligence Forge → Delivery Commander → Deal Engine → Agent 007 | What changed, what committed work is at risk, and what pre-award action is next? |
| Weekly | Intelligence Forge → Delivery Commander → Deal Engine → War Architect → Systems Blacksmith → Agent 007 | What evidence changed the plan, what can ship, which opportunity deserves attention, which three campaigns remain active, and what repetition is ready to systemize? |
| Monthly | Intelligence Forge → Delivery Commander → Deal Engine → Systems Blacksmith → War Architect → Agent 007 | Which sources and controls are stale, what produced value, what burden should be retired, and which professional outcomes govern the next month? |

APEX mode guidance:

- Intelligence Forge uses `intake_normalization` for new evidence and `source_replay` for cursor recovery before creating briefs.
- Delivery Commander receives exactly one of `delivery_control`, `technical_qa`, `quantity_delta`, or `cost_evidence`; Agent 007 splits mixed work.
- Deal Engine separates `pipeline_triage`, `reactivation`, and `proposal_control`.
- War Architect uses `operating_campaign`, `career_integration`, or `delegation_topology`.
- Systems Blacksmith enters only after stable repetition or a recurring material-error pattern is evidenced.

## JEOS cadence

| Cadence | Order | Required decision |
|---|---|---|
| Daily | Life Architect → Energy Director → Momentum Engine → Agent 007 | What matters today, what capacity limits placement, and what is the smallest feasible action queue? |
| Weekly | Reflection Forge → Energy Director → Life Architect → Momentum Engine → Lifestyle Systems Builder → Agent 007 | What was learned, what capacity is real, what direction changes, what actions follow, and what repeated friction may deserve a system? |
| Monthly | Reflection Forge → Energy Director → Lifestyle Systems Builder → Life Architect → Momentum Engine → Agent 007 | Which patterns held, what maintenance burden exists, what life direction changes, and which next actions begin the new month? |

JEOS routing precedence:

- Life Architect chooses outcomes and anchors.
- Energy Director constrains capacity and optional placement; it does not choose priorities.
- Momentum Engine creates next actions and restart rules after priorities are known.
- Reflection Forge labels lived evidence and tests hypotheses; it does not diagnose.
- Lifestyle Systems Builder enters only for stable recurring life administration or a material repeat error.
- Private constraints must match the destination agent's exact manifest `constraint_type:use_mode` profile.

## Completion and lifecycle

The local validation harness proves only that manifests parse, v2.1 packets satisfy the contract, and opposite-brain probes fail closed. It does not invoke a named agent, call a connector, complete a real mission, or promote a specialist.

Promotion requires one controlled real mission for every material mode, recorded accuracy and boundary evidence, verified runtime connector isolation, and readback for any mutation. A versioned native-agent change must remove `read-only` before a specialist may ever hold its own writer lease.
